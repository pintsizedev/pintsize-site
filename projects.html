Budget System A budget system written in Python using Django

This is a budget system written in Python using the Django web framework. It's currently on hold whilst I focus my web efforts on getting my blog up and running, and while I debate whether or not to continue it. I enjoy Python, very much so. But I'm now wondering whether this web app would be better done in an external application, rather than a web one. There are pros and cons for each, so I need to think about it.

My aim for this is essentially just to have a system where a user can input all transactions, and set up a budget with a certain amount per week. And the system will figure out whether they are above, or below their budget. Ideally you'd eventually be able to display graphs of your progress, but I'd want to get the core system working first.

Currently, the system supports adding new budgets, and adding transactions. As well as creating a user, with username and password. It uses a SQLite3 database, and the CSS is Bootstrap.

I chose Django as the web framework simply because it was in Python, and I heard it did some cool things, such as the automatically generated admin pages (Which still amaze me), and the pre-made user model.

The code for this can be found on GitHub here.

A* Pathfinding An A* pathfinding algorithm written in Lua

During the development of my game, I realised at some point I would probably need some form of pathfinding system. I could have done it badly, but why not try to learn a proper algorithm?

What I ended up making was a small program that visulised my a* algorithm using Löve, with the ability to change the start and end points. It would then display the shortest path taken to get from point A, to point B.

Although consider this project completed, I will revisit it in the future when I decide to finally include it in my game. I will probably end up adding a couple extra features into it, and I will turn it into an actual module, as opposed to a self contained program.

The code for this project can be found on GitHub here

Simple Booking System A simple booking system built with Ruby on Rails

Let me just start by saying that this project was my A2 coursework for my Computer Science A Level. And it's bad.

This was my first look at the world of web frameworks. We were given a task, to develop a system that used at least 2 database tables. And our teacher reccommended that we use Php. The problem with this was that we could only develop it on the school computers, not at home. And that we couldn't view the logs in order to debug it easier. So, I decided to do it in Ruby on Rails.

At the start of this, I had no knowledge of Ruby. I had never used a web framework. The term MVC meant nothing to me. My experience with MySQL was very little. And I had to learn this all on my own, as my teacher also knew as much as I did about Rails.

This code is bad. It really is. The term 'fat model, skinny controller' doesn't even apply in the slightest to this. So much of my controller code, should be in the model. It's a mess, it's most likely filled with mistakes due to my misunderstanding of Ruby.

The UI is also pretty bad. Looking back on it now, makes me sad. If I hadn't have used Bootstrap, I can't even begin to imagine how awful it would look.

I hope to revisit the idea of this one day, just to prove to myself that I can do it correctly. There are a few 'features' of that code that I did because I didn't want to deal with certain things to code because they wouldn't get me marks, such as the ability to book different lengths of time, other than a week. So one day, I will re-do this. Maybe not in RoR, but I will do this, and I will do it right.

The code for this can be found on GitHub here.
